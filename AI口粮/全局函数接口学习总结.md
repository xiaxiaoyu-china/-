## 神途游戏客户端全局函数接口学习总结

### 一、概述

全局函数是神途游戏客户端中用于处理系统级和游戏逻辑事件的回调函数集合。这些函数可以在特定事件触发时被自动调用，允许开发者自定义游戏行为和用户体验。

**适用平台**：根据函数类型不同，支持手游（2D/3D）、端游（2D/3D）或H5

---

### 二、接口分类与使用方法

#### 1. 场景与目标检测函数

| 接口名 | 中文名 | 参数 | 功能描述 | 示例代码 |
|--------|--------|------|----------|----------|
| BehindWall | 场景遮挡触发时回调 | _Value (int) | 当角色被场景遮挡时触发，1表示触发，0表示结束 | `function BehindWall(_Value) if _Value == 1 then dbg("角色被遮挡") end end` |
| CheckNpcClicked | 判断是否点中目标 | _GUID, _Type, _ID, _PosX, _PosY, _ClickX, _ClickY | 判断是否点中了可对话的角色（NPC或怪物） | `function CheckNpcClicked(_GUID,_Type,_ID,_PosX,_PosY,_ClickX,_ClickY) return _Type == 1 end` |

#### 2. 系统消息与界面定制函数

| 接口名 | 中文名 | 参数 | 功能描述 | 示例代码 |

| GetAttrRefNameCustom | 自定义商城购买前确认消息中的消费类型文字 | AttrType | 自定义商城购买提示面板中消费类型的文字 | `function GetAttrRefNameCustom(AttrType) return AttrType == 1 and "测试元宝" or "金币" end` |
| GetClientTeamHeaderImageID | 设置角色头像显示的资源 | headerIndex, gender | 更改游戏中程序默认定义的角色头像图片资源 | `function GetClientTeamHeaderImageID(headerIndex, gender) return 1000 + headerIndex end` |

### 三、实际应用示例

#### 示例1：自定义场景遮挡检测

```lua
-- 场景遮挡触发时回调
function BehindWall(_Value)
    if _Value == 1 then
        -- 角色被场景遮挡
        dbg("角色被场景遮挡，触发遮挡效果")

        -- 可以在这里添加遮挡时的特殊效果
        CL:SysMsg_SendPromptMsg("角色被场景遮挡")
    else
        -- 角色不再被场景遮挡
        dbg("角色离开遮挡区域")
        CL:SysMsg_SendPromptMsg("角色离开遮挡区域")
    end
end

-- 该函数无需主动调用，会在场景遮挡事件触发时自动执行
```

#### 示例2：自定义NPC点击检测

```lua
-- 判断是否点中目标
function CheckNpcClicked(_GUID, _Type, _ID, _PosX, _PosY, _ClickX, _ClickY)
    dbg(string.format("点击目标信息：GUID=%s, 类型=%d, ID=%d, 位置=(%d,%d), 点击=(%d,%d)",
        _GUID, _Type, _ID, _PosX, _PosY, _ClickX, _ClickY))

    -- 仅允许点击特定ID的NPC
    if _Type == 1 and (_ID == 1001 or _ID == 1002) then
        dbg("允许点击该NPC")
        return true
    end

    -- 禁止点击特定类型的怪物
    if _Type == 2 and _ID == 2005 then
        dbg("禁止点击该怪物")
        CL:SysMsg_SendWarnMsg("该怪物不可点击")
        return false
    end

    -- 默认允许点击
    return true
end

-- 该函数应添加到LogicFunction.lua文件中
```

#### 示例3：自定义系统消息样式

```lua
-- 自定义系统消息的控件样式和动画类型
function CreateSysAnnItem(parent, msg, param)
    -- 获取屏幕尺寸
    local screen_width = CL:GetScreenWidth()
    local screen_height = CL:GetScreenHeight()

    -- 创建自定义系统消息界面
    local bg_width = 350
    local bg_height = 40

    -- 创建顶部装饰
    GUI:ImageCreate(parent, "TopEdge", 1853880000, -bg_width/2, -5)
    -- 创建背景
    local bg_handle = GUI:ImageCreate(parent, "Background", 1855700005, -bg_width/2, 0)
    if bg_handle ~= 0 then
        GUI:WndSetSizeM(bg_handle, bg_width, bg_height)
        GUI:ImageSetFitSize(bg_handle, true)
        GUI:ImageSetAlpha(bg_handle, 150)
    end
    -- 创建底部装饰
    GUI:ImageCreate(parent, "BottomEdge", 1853880000, -bg_width/2, bg_height - 5)

    -- 创建文本显示控件
    local text_handle = GUI:RichEditCreate(parent, "MessageText", -bg_width/2, 5, bg_width, bg_height - 10)
    if text_handle ~= 0 then
        GUI:WndSetEnableM(text_handle, false)
        GUI:RichEditAppendString(text_handle, msg)
    end

    -- 返回自定义配置
    local ret = {}
    ret.custom = true
    ret.moveHeight = 45 -- 消息向上平移距离
    ret.centerX = screen_width / 2
    ret.centerY = screen_height - 100 -- 显示在屏幕底部
    ret.localZOrder = 150 -- 显示层级
    ret.animationType = 2 -- 动画类型：2表示滑动出现

    return ret
end

-- 该函数应添加到LogicFunction.lua等文件中
```

#### 示例4：自定义商城消费类型文字

```lua
-- 自定义商城购买前确认消息中的消费类型文字
function GetAttrRefNameCustom(AttrType)
    dbg("商城消费类型："..AttrType)

    local attrNames = {
        [1] = "测试元宝",
        [2] = "绑定元宝",
        [3] = "活跃度",
        [4] = "游戏币",
        [5] = "绑定游戏币"
    }

    local customName = attrNames[AttrType]
    return customName or "未知属性"
end

-- 该函数应添加到GameMall.lua文件中
```

### 三、学习建议与最佳实践

#### 1. 学习路径

1. **基础阶段**：学习场景与目标检测函数，了解事件触发机制
2. **进阶阶段**：学习系统消息与界面定制函数，掌握界面外观和行为控制
3. **高级阶段**：学习如何组合多个全局函数，实现复杂的游戏逻辑

#### 2. 最佳实践

- 将全局函数添加到适当的文件中，如LogicFunction.lua或GameMall.lua
- 对每个全局函数添加详细的注释，说明其功能和使用方法
- 定期测试全局函数的效果，确保其行为符合预期

#### 3. 常见使用场景

- 游戏内的交互优化，如NPC对话条件限制
- 系统消息的视觉定制，提升用户体验
- 角色状态反馈，如遮挡提示
- 商城和交易系统的个性化
